2. Steps

System is booted in 64-bit x64 UEFI mode: 

$ cat /sys/firmware/efi/fw_platform_size
64

Partition table: 

/dev/sda1  300M       ef  EFI (FAT 32)
/dev/sda2  1G         82  Linux swap
/dev/sda3  remaining  83  Linux        (encrypted)

3. Disk Encryption

I use disk encruption with passphrase and only encrypt my root partion. I do not encrypt my boot partition.

Load module dm_crypt: 

$ modprobe dm_crypt

$ cryptsetup luksFormat /dev/sda3
$ cryptsetup open /dev/sda3 root

Once opened, the root partition device address would be /dev/mapper/root instead of the partition (e.g. /dev/sda3).

In order to write encrypted data into the partition it must be accessed through the device mapped name.

To close the LUKS container, unmount the partition and do: 

$ cryptsetup close root

Make and format our filesystems. The UEFI partition must be FAT 32. 

$ mkfs.fat -F 32 /dev/sda1
$ mkswap /dev/sda2
$ mkfs.btrfs -L root /dev/mapper/root

Mount our partitions. The compression argument used here will be used by genfstab below… but of course we can still edit /etc/fstab later. 

$ mount -o compress=zstd:3 /dev/mapper/root /mnt
$ mount --mkdir /dev/sda1 /mnt/boot
$ swapon /dev/sda2

Bootstrap: 

$ pacstrap -K /mnt base linux linux-firmware

Generate our /etc/fstab: 

$ genfstab -U /mnt >> /mnt/etc/fstab

Before chrooting into our new system, let's create our btrfs subvolumes and set them to mount: 

$ btrfs subvol create /mnt/@
$ btrfs subvol create /mnt/@home
$ btrfs subvol create /mnt/@.snapshots
$ btrfs subvol create /mnt/@var_log

Add them to our /etc/fstab: 

UUID=xyz  /            btrfs  rw,relatime,compress=zstd:3,space_cache=v2,subvol=@            0  1
UUID=xyz  /home        btrfs  rw,relatime,compress=zstd:3,space_cache=v2,subvol=@home        0  0
UUID=xyz  /.snapshots  btrfs  rw,relatime,compress=zstd:3,space_cache=v2,subvol=@.snapshots  0  0
UUID=xyz  /var/log     btrfs  rw,relatime,compress=zstd:3,space_cache=v2,subvol=@var_log     0  0

The reason why I use fs_passno 0 (except for the /) is https://man.archlinux.org/man/fsck.btrfs.8.en

4. Chrooted with arch-chroot

4.1. Network

Install: 

$ pacman -S networkmanager

Enable: 

$ sysctl enable NetworkManager.service

4.2. Initial ramdisk

Add encrypt to HOOKS in /etc/mkinitcpio.conf (after udev, and between block and filesystems). Refresh the initial ramdisk: 

$ mkinitcpio -P

4.3. GRUB

Install grub and efibootmgr: 

$ pacman -S grub efibootmgr

Before running grubs's installation script, set the default btrfs volume to be the root of our filesystem so we don't have to pass it as a kernel argument:

(Install btrfs-progs if needed.) 

$ btrfs subvolume set-default 5 /

Run grub's installation script: 

$ grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB

Configure kernel arguments in /etc/default/grub. In my GRUB_CMDLINE_LINUX_DEFAULT I add: 

resume=<UUID-swap-partition>
root=/dev/mapper/root
cryptdevice=UUID=<UUID-root-partition>:root
crypto=::::

We can find the UUIDs with: 

$ ls -l /dev/disk/by-uuid

Refresh grub config file: 

$ grub-mkconfig -o /boot/grub/grub.cfg

5. Checks after first reboot

Swap status: 

$ swapon --show

Internet connection: 

$ ping archlinux.org

Root partition is encrypted (TYPE=crypto_LUKS): 

$ blkid /dev/sda*

Filesystem is btrfs in our encrypted root: 

$ btrfs filesystem show /dev/mapper/root

Our btrfs subvolumes: 

$ btrfs subvol list /

6. Troubleshooting with an arch live cd

Boot from the cd.

Decrypt our root partition: 

$ cryptsetup open /dev/sda3 root

Mount root and boot (assuming the EFI partition is /dev/sda1): 

$ mount /dev/mapper/root /mnt
$ mount /dev/sda1 /mnt/boot

Chroot: 

$ arch-chroot /mnt

To exit: 

$ exit
$ umount -R /mnt
